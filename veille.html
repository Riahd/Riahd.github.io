Introduction

🔍 Qu'est-ce que la veille informatique ?

La veille informatique consiste à surveiller en continu les évolutions technologiques et les nouvelles menaces dans le domaine de l'IT. En cybersécurité, elle est indispensable : de nouvelles vulnérabilités sont découvertes chaque jour et les outils utilisés pour les tester évoluent en permanence.

🎯 Pourquoi c'est important pour un pentester ?

Mon objectif professionnel est de devenir pentester (testeur d'intrusion). Ce métier consiste à simuler des attaques sur des systèmes pour en identifier les failles avant qu'un vrai attaquant ne les exploite. Pour être efficace, un pentester doit connaître les derniers outils disponibles, les nouvelles vulnérabilités et les techniques d'exploitation actuelles.

C'est pourquoi j'ai centré ma veille sur Kali Linux, la distribution de référence pour les tests d'intrusion. Suivre son évolution me permet de comprendre comment les outils de pentest s'adaptent aux nouvelles menaces.

Axe de veille — Kali Linux et les outils de pentest

L'outil central de ma veille

🐉 Pourquoi Kali Linux ?

Kali Linux est une distribution Linux spécialement conçue pour les tests d'intrusion et l'audit de sécurité. Elle regroupe plus de 600 outils de pentest préinstallés (Nmap, Metasploit, Burp Suite, Wireshark, etc.) et est maintenue par Offensive Security, une référence dans le domaine de la cybersécurité.

À chaque nouvelle version, Kali intègre de nouveaux outils qui reflètent l'évolution des techniques d'attaque. Suivre ces mises à jour permet de comprendre quelles sont les menaces actuelles et comment les pentesters s'y adaptent.

Exemple d'évolution — Nouveaux outils Kali Linux

Comment les outils de pentest s'adaptent aux nouvelles menaces

📋 Kali Linux 2025.1a (mars 2025)

En analysant cette version, j'ai identifié deux ajouts significatifs :

  • hoaxshell — un générateur de reverse shell Windows qui utilise le protocole HTTP/HTTPS pour établir une connexion discrète avec la machine cible. Cet outil illustre l'évolution vers des techniques de contournement plus furtives : le trafic ressemble à du trafic web normal, ce qui le rend difficile à détecter par les pare-feux. Un pentester utilise ce type d'outil pour tester si les défenses réseau détectent les connexions sortantes suspectes.
  • NetHunter CAN Arsenal — un module permettant le pentest de véhicules connectés (car hacking) depuis un smartphone. Cela montre que le pentest s'étend à de nouveaux domaines (IoT, automobile). Les outils de Kali suivent cette tendance pour permettre aux professionnels de tester ces nouveaux environnements.

📋 Kali Linux 2026.1 (mars 2026)

Version la plus récente au moment de ma veille :

  • Kernel 6.18 — nouveau noyau Linux avec corrections de sécurité et support matériel étendu, permettant d'utiliser Kali sur davantage de machines.
  • Mises à jour des outils existants — Nmap, Metasploit, Burp Suite et d'autres outils de pentest ont reçu des mises à jour pour prendre en charge de nouvelles techniques d'exploitation.

💡 Le lien avec le pentest

Chaque nouvelle release de Kali ajoute ou met à jour des outils en réponse aux nouvelles techniques d'attaque. Un pentester qui ne suit pas ces évolutions risque de ne pas connaître les outils adaptés aux menaces actuelles. C'est exactement le rôle de la veille : rester informé pour être efficace lors des audits de sécurité.

Exemple de vulnérabilité — Grafana (XSS)

CERTFR-2025-AVI-0597 — Une faille exploitable avec les outils de Kali

⚠️ La vulnérabilité

CERTFR-2025-AVI-0597 — Multiples vulnérabilités dans Grafana (18 juillet 2025)

De multiples vulnérabilités ont été découvertes dans Grafana, un outil de visualisation de données que j'ai moi-même déployé dans mon Projet 2 (centralisation des logs).

  • Injection de code indirecte à distance (XSS) — un attaquant peut injecter du code malveillant dans l'interface web, menant au vol de session ou à l'exécution d'actions au nom de la victime.
  • Contournement de la politique de sécurité — un attaquant peut contourner les mécanismes de contrôle d'accès.

Systèmes affectés : Grafana versions 11.3.x, 11.4.x, 11.5.x, 11.6.x et 12.0.x antérieures aux versions corrigées.

CVE associées : CVE-2025-6197, CVE-2025-6023.

Source : CERT-FR — CERTFR-2025-AVI-0597

🐉 Le lien avec Kali Linux

Ce type de vulnérabilité (XSS) peut être testé et exploité avec des outils présents dans Kali Linux. Par exemple :

  • Burp Suite — permet d'intercepter les requêtes HTTP et d'injecter des payloads XSS pour vérifier si l'application est vulnérable.
  • XSSer — outil automatisé de détection et d'exploitation de failles XSS, préinstallé dans Kali.

Lors d'un audit de sécurité (pentest), un testeur utiliserait ces outils pour vérifier si le Grafana déployé est vulnérable avant qu'un attaquant réel ne le fasse. C'est ce lien direct entre vulnérabilité publiée et outil de test qui rend la veille sur Kali Linux indispensable.

Exemple réel — Cyberattaque du SIA (Système d'Information sur les Armes)

Fuite massive de données sensibles en France

📌 Contexte

Une cyberattaque récente a visé le Système d'Information sur les Armes (SIA) en France. Selon les sources disponibles (OSINT et réseaux sociaux spécialisés en cybersécurité), les données sensibles de plus de 60 000 personnes auraient été compromises et mises en vente sur des forums clandestins.

Les informations exposées concernent notamment :

  • Identités — nom, prénom des détenteurs
  • Adresses — domiciles des personnes fichées
  • Informations liées à la détention d'armes — données critiques pouvant mettre en danger la sécurité des personnes

⚠️ Analyse de l'attaque

Fuite de données — Système d'Information sur les Armes (SIA)

Cette attaque met en évidence plusieurs problèmes majeurs :

  • Mauvaise sécurisation des accès — les mécanismes de protection du système étaient insuffisants.
  • Absence ou faiblesse du MFA — l'authentification multifacteur, mesure de base, n'était pas correctement implémentée.
  • Exposition de données critiques — des informations liées à la détention d'armes, particulièrement sensibles, étaient accessibles.

Conséquences potentielles : risques directs pour la sécurité physique des personnes fichées, exploitation des données à des fins criminelles (cambriolages ciblés, usurpation d'identité).

🐉 Le lien avec Kali Linux et le pentest

Ce type d'attaque peut être rapproché des techniques utilisées en pentest. Les outils présents dans Kali Linux permettent de simuler ces scénarios pour identifier les failles avant qu'un attaquant réel ne les exploite :

  • SQLmap — outil automatisé de détection et d'exploitation de failles d'injection SQL, utilisé pour tester si une base de données est accessible via des entrées non sécurisées.
  • Burp Suite — permet d'analyser les requêtes HTTP et d'identifier les vulnérabilités web (authentification faible, failles d'accès).
  • Hydra — outil de brute-force pour tester la robustesse des mots de passe et détecter l'absence de MFA.
  • Nikto — scanner de vulnérabilités web qui identifie les configurations insécurisées sur les serveurs exposés.

Si un pentester avait audité le SIA avec ces outils, il aurait pu identifier les failles d'accès et l'absence de MFA avant que l'attaque ne se produise. Cette veille me permet de comprendre comment des failles réelles sont exploitées et comment les outils de Kali servent à les prévenir.
